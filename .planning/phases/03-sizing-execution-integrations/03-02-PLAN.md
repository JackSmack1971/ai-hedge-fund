<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| state["data"]["meta_label_outputs"] → portfolio filtering | Label strings and size_multiplier floats sourced from meta_labeler LLM output; corrupt values could enable or suppress trades incorrectly |
| allowed_actions_full post-filter | Modifying the dict that controls what the LLM is allowed to pick; must not accidentally open restricted actions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | portfolio_manager.py size_multiplier read | mitigate | Clamp with max(0.0, min(1.0, float(meta.get("size_multiplier", 1.0)))) — prevents negative or >1.0 multipliers from expanding position beyond risk manager ceiling |
| T-03-06 | Elevation of Privilege | suppress bypass via missing allow_trade key | accept | meta.get("allow_trade", True) defaults to True — absent key means allow trade (safe default); suppress only fires when allow_trade is explicitly False |
| T-03-07 | Elevation of Privilege | hold_only: sell/cover accidentally removed | accept | Filter only removes "buy" and "short" keys; "sell" and "cover" are preserved — incorrect removal would be caught by TestMetaLabelHoldOnlyFilter::test_hold_only_existing_long |
| T-03-08 | Denial of Service | suppress + existing long position zeroes allowed actions | accept | By design (D-32): suppress means no trading at all. The risk is that a corrupt suppress label blocks legitimate exits — mitigated by the neutral-fallback test (Test 8) and the default allow_trade=True when key is absent |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 3 |
</threat_model>
